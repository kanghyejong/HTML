[WEB1 - 15. 웹사이트 완성]
링크는 실 본드 같은 역할을 한다. 만든 웹페이지들을 엮어서 웹사이트를 만들 수 있다.
소제목을 이용하여 다른 웹페이지로 링크를 걸어 이동한다.

[WEB1 - 16. 원시웹]
인터넷 안에 웹이 존재(포함관계) 인터넷은 1960년도 웹은 1990년도에 나왔다.
웹의 등장으로 인터넷의 확산이 이루어졌다.

[WEB1 - 17. 인터넷을 여는 열쇠: 서버와 클라이언트]
웹브라우저와 웹서버가 각각 깔려진 컴퓨터 두개 사이의 주소를 입력하여 정보를 주고 받았다. 
그렇게 정보를 요청하는 웹 클라이언트 응답하는 웹 서버라고 불리게 된다.

[WEB1 - 18. 웹호스팅 : github page]
제공되는 서버를 이용할 수 있다. 대표적으로 github가 있다. github를 인터넷처럼 사용하여 
github에 올려놓은 html 파일들로 연결되어 들어갈 수 있는 웹사이트를 만들 수 있다.
https://kanghyejong.github.io/MoGakCo-WebProgramming/나만의 웹사이트가 생긴다.

[WEB1 - 19. 웹서버 운영하기]
가장 많이 사용되고 있는 아파치라는 서버를 깔아본다.

[WEB1 - 19.1.1.. 웹서버 설치(윈도우)](20)
아파치를 윈도우에 직접 깔기는 힘드니 bitnami를 깔아서 아파치를 사용한다.

[WEB1 - 19.1.2. 웹서버와 http (윈도우)](21)
http://127.0.0.1/을 이용하여 자신의 컴퓨터에 웹브라우저와 웹서버 모두 깔려있을 때 
웹브라우저가 웹서버의 정보들을 불러올 수 있도록 한다. 127.0.0.1이 자신의 컴퓨터 서버에 
정보를 불러오도록 하는 주소(http - Hyper Text Trnsfer Protocol 웹브라우저와 웹서버간의 통신)

[WEB1 - 19.1.3. 웹서버와 웹브라우저의 통신(윈도우)](22)
서버가 저장되어있는 컴퓨터의 IP주소를 이용하여 주소에 접근할 수 있다.
서버가 저장되있는 컴퓨터와 다른 컴퓨터를 이용하여 IP주소를 입력하고 파일을 입력하면 
똑같은 웹페이지가 나올 것이다. 대신 같은 네트워크를 사용하고 있어야한다.
비트나미를 이용하여 접속 하는것은 실패했다. 다음 모임에서 다시 한번 시도 해봐야겠다.
